# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✓         |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Email **dollarbone@gmail.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You'll receive an acknowledgement within 48 hours and a resolution timeline once the issue is assessed. Critical vulnerabilities will be patched on a best-effort basis as quickly as possible.

Public disclosure is coordinated after a fix is available.
