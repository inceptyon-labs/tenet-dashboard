## Design Context

### Users
Solo developer monitoring their own projects after running code health audits via Claude Code. Views the dashboard to check scores, drill into findings, track trends over time. Context: focused work session, often late-day review.

### Brand Personality
Precise. Dense. Engineered. — Like a well-built instrument panel you trust implicitly.

### Aesthetic Direction
- **Tone**: Mission control — serious, information-rich, data-forward
- **References**: Linear, Raycast — polished dev tools with tight typography, keyboard-first design sensibility
- **Anti-references**: Generic admin templates, Grafana's visual noise, anything that looks like Bootstrap
- **Theme**: Dark — this is an evening/focused-work tool

### Design Principles
1. **Density over decoration** — every pixel earns its place; no filler
2. **Data is the design** — scores, trends, and findings ARE the visual interest; don't compete with them
3. **Quiet confidence** — refined details that reward close attention, not flashy effects
4. **Typographic hierarchy** — use type weight, size, and family to create structure, not boxes and borders
